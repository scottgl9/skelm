# Contributing

skelm is built in the open. Contributions — bug reports, design feedback, documentation fixes, and code — are welcome.

The canonical contributor guide and security policy live in `.github/` so GitHub's UI surfaces them as the "Contributing" prompt on issues/PRs and the "Report a vulnerability" tab:

- [Contributing guide](https://github.com/scottgl9/skelm/blob/main/.github/CONTRIBUTING.md) — prerequisites, branching, commits, PR checklist.
- [Security policy](https://github.com/scottgl9/skelm/blob/main/.github/SECURITY.md) — supported versions, threat model, vulnerability reporting.

The release flow stays in docs:

- [Publishing](./PUBLISHING.md) — how releases are cut and pushed to npmjs.org / GitHub Packages.
